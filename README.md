# Tahder Mobile

تطبيق Tahder هو مساعد ذكي للمعلم يهدف إلى اختصار وقت التحضير والمتابعة اليومية. المشكلة التي نحاول حلها أن المعلم قد يقضي ساعات طويلة في التحضير، البحث عن ملفات، توزيع المنهج، رصد المتابعة، وتجهيز التقارير. هدف التطبيق أن يحول هذه الأعمال إلى تدفق سريع وواضح من الجوال.

## الفكرة الأساسية

بدل أن يكون التطبيق مجرد مكتبة روابط أو ملفات، يكون لوحة عمل يومية للمعلم:

- يعرف جدول المعلم وحصص اليوم.
- يقترح التحضير المناسب لكل درس.
- يربط التحضير بالواجب، أوراق العمل، والأنشطة.
- يختصر رصد الحضور والمشاركة.
- يجهز التقارير والكشوفات والملفات القابلة للتصدير.
- يساعد المعلم في التعامل مع المنصات الرسمية مثل مدرستي ونور بدون إدخال متكرر.

## اتجاهنا المختلف

نريد أن نحافظ على شمولية الخدمات، لكن نطور التجربة نفسها. الفكرة ليست أن نعرض عشرات الأزرار للمعلم، بل أن نبدأ من احتياجه اليومي:

ما الدرس القادم؟ ما التحضير المطلوب؟ هل يوجد واجب مناسب؟ هل تم رصد الطلاب؟ هل التقرير جاهز؟

هذا يعني أن التطبيق يجب أن يكون:

- أسرع من الإدخال اليدوي.
- أقل ازدحاماً من صفحات الخدمات الطويلة.
- مبنياً حول اليوم الدراسي وليس حول قائمة ملفات.
- ذكياً في الاقتراح، النسخ، التوليد، والتصدير.
- مريحاً للجوال لأن المعلم يستخدمه داخل الفصل.

## وحدات المنتج

الهدف: فتح التطبيق يعطي المعلم صورة فورية عن يومه.

المطلوب:

- حصص اليوم مرتبة حسب الوقت.
- حالة كل حصة: تحتاج تحضير، جاهزة، تحتاج حضور، تحتاج واجب.
- زر تحضير سريع للحصة القادمة.
- نسبة إنجاز اليوم.
- تنبيهات مختصرة للواجبات، الغياب، والتقارير.
- اختصارات: التحضير، الحضور، المحتوى، التقارير.

### 2. التحضير الذكي

- اختيار المرحلة، الصف، المادة، الفصل الدراسي، الوحدة، والدرس.
- توليد تلقائي لعناصر التحضير:
- الأهداف.
- نواتج التعلم.
- التمهيد.
- الاستراتيجيات.
- الوسائل التعليمية.
- الأنشطة.
- التقويم.
- الواجب.
- الإغلاق.
- دعم تعديل سريع لكل حقل.
- نسخ التحضير لفصل آخر.
- حفظ كقالب شخصي.
- الاستفادة من بنك تحضير المعلمين لاحقاً.
- تصدير أو نسخ التحضير إلى منصات خارجية.

### 3. الواجبات والأنشطة

الهدف: ربط الواجب بالدرس مباشرة.

المطلوب:

- اقتراح واجب عند حفظ التحضير.

## تجربة المستخدم المطلوبة

التطبيق يجب أن يكون عملياً وهادئاً. المعلم لا يحتاج صفحة تسويقية داخل التطبيق، بل يحتاج أدوات جاهزة للعمل:

- أول شاشة هي لوحة اليوم.
- كل حصة لها إجراء واضح.
- كل إجراء يأخذ أقل عدد ممكن من الخطوات.
- النصوص قصيرة وواضحة.
- الواجهة عربية وRTL من البداية.
- الجوال هو الأولوية.
- لا نكثر الألوان بدون معنى.
- لا نضع عشرات الخيارات في شاشة واحدة.
- كل ميزة كبيرة تبدأ بتدفق بسيط ثم تتوسع.

## ربط Supabase

تم تجهيز أساس قاعدة بيانات مشترك للموقع وإضافة Chrome داخل مجلد `supabase`.

### ما تغطيه القاعدة

- الحسابات والملفات الشخصية.
- الخطط والاشتراكات والتحقق من صلاحية الوصول.
- الأجهزة المرتبطة وإعدادات الإضافة.
- فهرس المناهج والكتب.
- تحاضير الدروس وسجل التوليد الذكي.
- الفصول والطلاب.
- الحضور والغياب والمشاركة.
- الواجبات والأسئلة والتسليم والتصحيح.
- أوراق العمل التفاعلية.
- توزيع المنهج على الأسابيع.
- سجل التصدير ومهام المزامنة مع المنصات.
- سجل النشاط للمراجعة والدعم.

### خطوات الإعداد لأول مرة

1. أنشئ مشروعاً جديداً في [Supabase](https://supabase.com/dashboard).
2. افتح `SQL Editor` وشغل ملفات الترحيل بالترتيب:
   - `supabase/migrations/202606020001_initial_tahder_schema.sql`
   - `supabase/migrations/202606020002_application_modules.sql`
   - `supabase/migrations/202606020003_data_api_grants.sql`
   - `supabase/migrations/202606020004_curriculum_library.sql`
3. من `Project Settings > API Keys` انسخ:
   - رابط المشروع.
   - `Publishable key` العام فقط.
4. ضع الرابط والمفتاح العام في الملفين:
   - `extension/config.js`
   - `extension-site/config.js`
5. لا تضع `Secret key` أو `service_role` داخل الموقع أو الإضافة نهائياً.
6. أنشئ مستخدماً تجريبياً من `Authentication > Users`.
7. انسخ رقم المستخدم وأضف له اشتراك تجربة من `SQL Editor`:

```sql
insert into public.subscriptions (user_id, plan_id, status, ends_at)
values ('USER_UUID', 'trial', 'trial', now() + interval '7 days');
```

### تشغيل التوليد الذكي

نقطة التوليد موجودة في `supabase/functions/generate-preparation`. يجب نشرها كـ Edge Function وإضافة الأسرار من لوحة Supabase أو CLI:

```bash
supabase secrets set OPENAI_API_KEY=YOUR_OPENAI_KEY
supabase secrets set OPENAI_MODEL=gpt-5-mini
supabase functions deploy generate-preparation
```

مفتاح OpenAI يبقى في الخادم فقط ولا يصل إلى إضافة Chrome أو الموقع.

### وضع العرض المحلي

إذا كانت ملفات `config.js` فارغة، تستمر صفحات `localhost` في وضع demo لتطوير الواجهة. عند إضافة بيانات Supabase يصبح تسجيل الدخول والتحقق من الاشتراك حقيقياً تلقائياً.
